Inventory Management System

How the Application Works

Loading the home screen

  1. localhost:3000 is visited, the node js server processes the GET request of a root path. The server then sends the index.html file to the client.

  2. When the browser receives the index.html file, it executes javascript which performs an AJAX request through jquery, to receive a list of item IDs from the server.

  3. Another request is then performed to get the specific data of the first item in this list. When the arrow buttons are used, this list is cycled through, and another request is made to get the data of the current item in the list.

Adding an Item

  1. When the add item button is pressed, the user is directed to a new page.

  2. When the fields are filled out and the submit button is pressed, a POST request is sent to the server containing the entered data.

  3. The server processes this request, inserting the new data into the database

Updating an Item

  1. When the update button is pressed, the user is directed to a new page.

  2. A request is sent to the server for the data of the current item being updated, and this data is inserted into the fields

  3. After the user edits these fields and clicks submit, the data is sent to the server in a PUT request

  4. The database entry is then updated for that item.

Deleting an Item

  1. When the delete button is pressed, a delete request is sent to the server with the ID of the current item.

  2. The server processes this request and deletes the item from the database

Searching for an Item

  1. When the user enters a name of a product into a field and presses search, this search term is sent to the server as a url parameter

  2. This is processed, and the database is queried

  3. If an item is found, the ID of this is sent back to the client.

Technologies Involved in the Project

Node.js

Node js is an open source javascript runtime environment that allows javascript to be run on a server.

JQuery

A javascript library which simplifies the use of AJAX, which is a process of retrieving data asynchronously from a server.

MongoDB

A document orientated database program that utilises JSON instead of traditional SQL.

Express

A backend web framework for building RESTful APIs
